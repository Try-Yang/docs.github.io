打开react源码，也就是对应的packages文件夹找到react文件夹中的index.js，最后发现，这个文件夹最终仅仅是帮我们返回了一堆的方法属性。并没有什么其它的复杂的逻辑，这和react的设计思想有关，它将所有的功能都拆分成一个个的工具库，有react这个库单独符合，这样极大的方便了调试以及问题的定位，react库本身只负责提供对应的api，具体的和渲染相关的实现由render来控制。

这里解答一个开篇的疑问？我们为什么没有使用到React这个变量，但是确需要在每一个模块的开头引入react呢。

这其实主要是为babel提供的，现在的react项目开发方式基本都是基于webpack的工程化模式，在开发过程中，babel会将我们代码中的jsx部分，转换为对应的 React.CreateElement的形式，所以一旦没有引入，那么就会导致函数调用错误，当然这个形式是可以通过babel来配置的

{
    "plugins": [
        [
            "@babel/plugin-transform-react-jsx",
          // 通过pragma可以控制对应的jsx需要调用的方法名，默认情况是React.CreateElement
            {
                "pragma": "h"
            }
        ]
    ]
}

React.CreateElement也是渲染过程中的一个核心方法，后续进行分析。

关于React Core主要弄明白一件事情，react本身将所有的功能api都抽离出来，他不负责提供任何和渲染相关的负责逻辑，仅仅是一堆功能API的集合。这么做的目的是为了完全的和render脱离，来实现更好的跨平台支持性。
