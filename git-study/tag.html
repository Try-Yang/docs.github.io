    # Tag

    标签管理也是git的一个很重要的功能，相比较于大量的commitId，标签无疑会清晰很多，因为开发功能的原因，我们再开发过程中一般都会提交很多次的 commit，虽然可以通过 rebase 进行 commit的合并，但是再大量使用的情况下还是会很多，所以引入了 tag，相比较 commitId，tag的可读性也很好。

    tag实质上就是一个 commitId的指针，但是它是不可移动的，我们可以再每一次要上线的 commitId上打上一个版本 tag v1.0.0之类的。这样我们可以很方便的去查看每次发布的详细情况

    # tag的使用

    # 查看tag
    git tag
v1.0
v2.0
    1
    2
    3
    # 创建tag

    轻量tag（只包含某个特定commit的引用）和附注tag（可以包含很多信息，同时也可以被校验，信息包含 打标签者，时间，邮件等等）

    # 附注标签
    git tag -a v1.0 -m "1.0版本上线了"

git tag
